yoom.pro

Сайт агентства недвижимости (новостройки Сочи): презентация компании, каталог жилых комплексов с фильтрами и картой, карточки ЖК и квартир, личный кабинет пользователя, приём заявок и защищённая админ-панель для просмотра заявок и пользователей.

Фронтенд

Вёрстка: семантический HTML5 (header, section, main, aside, nav, footer). Многостраничная структура: главная, каталог, карточка комплекса, квартиры, личный кабинет, вход/регистрация, админ-панель.

Стили: CSS3, каскад из нескольких файлов (styles.css, tablet.css, mobile.css, login.css, safari-fixes.css, critical.css). Flexbox и Grid, CSS-переменные, медиа-запросы для планшета (768px) и телефона (576px). Отдельный файл с фиксами для iOS/Safari (safe-area, tap-highlight, 100vh/dvh).

Интерактивность: ванильный JavaScript без фреймворков. Обработка событий, делегирование. Работа с DOM, историей браузера (history.replaceState) и URL (URLSearchParams). Запросы к API через Fetch с заголовками (Content-Type, X-Auth-Token, X-Admin-Hash).

Функциональные блоки: слайдер баннера на главной, быстрый поиск по количеству комнат, фильтры каталога (район, застройщик, цена, комнаты), сортировка, поиск по названию/адресу в шапке, карта каталога (Yandex Maps API), избранное, ипотечный калькулятор (первоначальный взнос, срок, расчёт платежа и переплаты), форма заявки с отправкой в API, модальные окна, мобильное меню (оверлей + боковая панель), редирект неавторизованных при переходе в кабинет и при добавлении в избранное.

Авторизация и личный кабинет

Регистрация и вход: форма с вкладками «Вход» и «Регистрация». Логин и пароль, валидация на клиенте и сервере. Поля с autocomplete для сохранения паролей и Face ID/Touch ID в Safari/iOS. После регистрации — опциональный шаг «Профиль» (имя, фамилия, телефон, email, аватар) с возможностью пропустить.

Сохранение сессии: токен и объект пользователя в localStorage, отправка токена в заголовке X-Auth-Token. Выход — вызов API logout и очистка localStorage.

Личный кабинет: избранные комплексы (список с картинкой, названием, ссылкой «Подробнее» и кнопкой «Удалить»), список заявок пользователя, раздел «Настройки» (аватар, имя, фамилия, телефон, email) с сохранением через API. Для неавторизованных кнопка «Выйти» заменяется на «Войти или зарегистрироваться» с редиректом на страницу входа.

Бэкенд (API)

Среда: PHP 7+, MySQL (mysqli), единая точка входа через init.php (конфиг, CORS-заголовки, соединение с БД, функции jsonOut, getAuthToken, getUserIdByToken).

Аутентификация: api/auth/register.php (POST: логин, пароль → проверка формата и уникальности, bcrypt-хеш, создание пользователя и сессии, возврат token и user). api/auth/login.php (POST: проверка пароля через password_verify, создание сессии). api/auth/logout.php (POST: инвалидация сессии по токену).

Профиль и аватар: api/user/profile.php (GET — данные профиля по X-Auth-Token; POST — обновление имени, фамилии, email, телефона, avatar_url). api/user/avatar-upload.php (POST, multipart: загрузка файла в uploads/avatars/, проверка типа и размера).

Избранное: api/favorites.php (GET — список избранного; POST — добавление по item_id, name, link, image; DELETE по item_id). Привязка к user_id по токену.

Заявки: api/requests.php (POST: создание заявки с полями имени, телефона, email, комментария). api/admin/requests.php (GET — список заявок для админки; PUT/POST — смена статуса). Защита заголовком X-Admin-Hash.

Безопасность: подготовленные запросы (prepare/bind_param) против SQL-инъекций; rate limiting по IP; проверка origin для CORS; хранение паролей в виде bcrypt-хеша; админ-эндпоинты доступны только при совпадении хеша из конфига.

Инфраструктура и деплой

Чистые URL через Apache mod_rewrite (.htaccess): редирект с .html на URL без расширения (301), маршрутизация запросов к соответствующим .html-файлам, сохранение query-строки (QSA).

База данных: таблицы users, sessions, favorites, requests, rate_limit. При необходимости в register/login выполняется ALTER TABLE для добавления полей login и password_hash.

Адаптив и UX

Три брейкпоинта: десктоп, планшет (768px), мобильный (576px). На мобильном: компактный хедер, поиск по центру, скрытие кнопки «Избранное» в шапке и блока «Быстрый поиск», мобильное меню с перехватом перехода в «Личный кабинет» для неавторизованных. Кнопки избранного для неавторизованных ведут на страницу входа с return-url.

Погода

Одностраничное приложение для портфолио: пользователь видит текущую погоду по своему местоположению, подробные показатели, восход/закат и прогноз на неделю с графиком температур.

Данные

Используется бесплатный API Open-Meteo (без ключа). Погода подставляется по геолокации браузера; при отказе или ошибке подставляется Москва.

Интерфейс

Блок «Сейчас»: температура, описание (ясно / облачно / дождь и т.д.), «Ощущается как … °C».

Блок «Детали»: сетка плиток — ощущается, влажность, ветер, давление, видимость, УФ-индекс, при осадках — текущие осадки (мм).

Блок «Световой день»: время восхода и заката для текущего дня.

Прогноз на 7 дней: список дней (сегодня / название дня и дата), иконка погоды, макс/мин температура, осадки при наличии.

График температуры на неделю: две линии (макс/мин) с заливкой, тултип по дням (Recharts).

Визуал и анимации

Фон и сцена меняются от времени суток и типа погоды: день — голубое небо, солнце или облака; дождь — серый фон и анимация капель; гроза — те же эффекты плюс вспышки «молнии»; снег — холодные тона и падающие снежинки; ночь при ясной погоде — тёмный градиент, луна и мерцающие звёзды; ночью при дожде/грозе — тёмный фон с дождём и молнией. Карточки и блоки оформлены в стиле стекла (полупрозрачный фон, размытие, границы).

Стек и деплой

React 18, Vite, Open-Meteo API, Recharts (график), CSS (анимации, адаптив). Адаптивная вёрстка под мобильные (отступы, сетки, типографика до ~480px), учтены safe-area для вырезов экрана.

Сборка через GitHub Actions, выкладка на GitHub Pages. Ссылка: scalevillain13.github.io/weather-app

Kanban Board

Мини-система управления задачами в формате Kanban: быстрый обзор задач по статусам, приоритизация и перемещение карточек между колонками.

Функционал

Три колонки: Backlog, In Progress, Done. Карточки можно создавать, редактировать, удалять и переносить между колонками drag-and-drop.

Добавлены приоритеты High / Medium / Low, поиск по задачам, фильтр по приоритету, быстрый перенос карточки кнопками и очистка колонки Done.

Состояние доски сохраняется в localStorage, поэтому после перезагрузки данные остаются.

React-часть

Проект собран на React с разделением на компоненты: колонка, карточка, форма создания и панель фильтров. Используются хуки useState и useMemo для управления состоянием и производительной фильтрации.

Drag-and-drop реализован на HTML5 Drag and Drop API с мгновенным обновлением состояния без перезагрузки страницы.

Технологии

React, Vite, CSS3, localStorage, HTML5 Drag and Drop API.

Деплой: GitHub Pages. Ссылка: scalevillain13.github.io/kanban-board

Blog API

REST API для блога на Laravel с авторизацией через токен (Sanctum). Полный CRUD постов и комментариев, фильтрация, пагинация и защита маршрутов.

Авторизация

Регистрация и логин через Laravel Sanctum. Токен возвращается при регистрации/логине и передаётся в заголовке Authorization: Bearer <token>. Есть эндпоинты для logout и получения текущего пользователя.

Посты

Полный CRUD: список (с пагинацией, поиском по заголовку, фильтром по статусу публикации), просмотр одного поста с комментариями, создание, обновление и удаление. Автоматическая генерация slug. Редактировать и удалять пост может только автор.

Комментарии

Список комментариев к посту (с пагинацией), создание и удаление. Удалить комментарий может только его автор.

Архитектура

Валидация: выделена в Form Requests (StorePostRequest, UpdatePostRequest, StoreCommentRequest).

Модели: Eloquent ORM с отношениями (hasMany, belongsTo), авто-генерация slug через booted() callback, массовое присваивание через fillable.

Ответы: единый JSON-формат { success, data, error } на всех эндпоинтах.

Стек

PHP 8.1+, Laravel 11, Laravel Sanctum, MySQL / SQLite, Eloquent ORM, Form Requests.

Circles

Компонент кругового прогресс-бара на Vue 3 (Composition API) без сторонних библиотек. Реализация на SVG и дополнительно на Canvas.

Возможности

Типы: circle (полный круг) и dashboard (дуга с разрывом снизу).

Состояния: in_progress, success, warning, error — с плавной анимацией при смене значения и статуса. В in_progress цвет заполнения от красного (0%) к зелёному (100%). В центре: процент или иконки (галочка / ! / крестик).

Технологии

Vue 3, Composition API, Vite. SVG и Canvas-вариант с теми же props, анимация через requestAnimationFrame. Адаптивная вёрстка.

Убийца Слизней

Геймплей

Технологии

Чистый JavaScript (без фреймворков), HTML5, CSS. Звуки, адаптивная вёрстка. Деплой на GitHub Pages.
